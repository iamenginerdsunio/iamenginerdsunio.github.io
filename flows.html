Code Flows

Understand the detailed flow of requests, data, and application logic in Flask-AppBuilder.

Code Flow Overview

Flask-AppBuilder processes user requests and events using Flask's routing and view mechanisms, SQLAlchemy for data management, and a modular MVC structure. Requests flow through views to interact with models, resulting in responses generated for the user.

Request Lifecycle

  1. User sends a request to the server (e.g., HTTP GET or POST).
  2. The Flask application routes the request using defined Blueprints.
  3. Views handle the request, invoking business logic and interacting with models.
  4. Models perform database operations using SQLAlchemy ORM.
  5. The view generates a response (HTML, JSON, etc.) and sends it back to the user.

Routing

Routes are defined using Flask's Blueprint system for modular routing.

class BaseView:
    def register_routes(self):
        blueprint = Blueprint('base', __name__)
        self.appbuilder.app.register_blueprint(blueprint)

Examples:

  • /modelview handled by ModelView.list for listing model data.
  • /api/create_item handled by api.create_item for creating a database item.

Data Flow

Data flows from user inputs through views to models, which utilize SQLAlchemy ORM for database interactions. After processing, responses are generated and returned to the user.
