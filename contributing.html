Contributing

Quick Setup

To set up the project locally, follow these steps:

  1. Clone the repository: git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Navigate into the project directory: cd LibManagement
  3. Create a virtual environment: python -m venv venv
  4. Activate the virtual environment:
    • On Windows: venv\Scripts\activate
    • On macOS/Linux: source venv/bin/activate
  5. Install dependencies: pip install -r requirements.txt
  6. Run the application: python app.py

Contribution Workflow

The recommended workflow for contributing is:

  1. Fork the repository on GitHub.
  2. Create a new branch for your feature or bug fix: git checkout -b feature-name
  3. Make changes in your branch and commit them with clear messages.
  4. Push your branch to your fork: git push origin feature-name
  5. Open a pull request against the main branch of the original repository.

Code Hygiene

Maintain high standards in your contributions by following these guidelines:

  • Write clear, modular, and well-documented code.
  • Adhere to Python's PEP 8 style guide.
  • Ensure proper use of Flask's best practices for routes and templates.
  • Update README.md if your changes affect setup or usage instructions.

Testing Your Changes

Testing is an integral part of contributions. Use the existing pytest suite to validate your changes:

  1. Ensure pytest is installed in your environment (included in requirements.txt).
  2. Run tests: pytest
  3. Write new tests in tests/test_app.py for any new functionality or changes you introduce.

Tests should cover all major routes and their expected behaviors, including CRUD operations for books, members, and loans.

Operational Notes

Keep the following points in mind while contributing:

  • Ensure compatibility with the existing Flask configuration and database models defined in extensions.py and models.py.
  • Any changes to templates should respect the layout structure defined in base.html.
  • Static assets in the static/ directory should be appropriately referenced in templates.
