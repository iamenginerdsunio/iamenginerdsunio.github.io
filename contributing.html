Contributing to Flask-AppBuilder

Welcome

Thank you for considering contributing to Flask-AppBuilder! We value all contributions, whether it's bug fixes, feature additions, or documentation improvements. By contributing, you are helping to improve a framework that simplifies web application development for everyone.

Please make sure to review our Code of Conduct to ensure a welcoming and collaborative environment.

Prerequisites

Before you start contributing, please ensure you have the following:

  • Python 3.7 or newer.
  • Flask installed (pip install flask).
  • A GitHub account to submit pull requests.
  • Basic understanding of Flask and Python.

Development Setup

Follow these steps to set up the Flask-AppBuilder repository for development:

  1. Clone the repository: 
    git clone https://github.com/dpgaspar/Flask-AppBuilder.git
  2. Install dependencies: 
    pip install -r requirements.txt
  3. Configure the environment variables: 
    export SQLALCHEMY_DATABASE_URI=sqlite:///app.db
export SECRET_KEY=mysecretkey
  4. Verify the installation: 
    flask run

If you encounter any issues, please consult the Troubleshooting guide.

Project Structure

Here's a quick tour of the repository:

  • flask_appbuilder/: Core application modules.
  • tests/: Unit tests organized by functionality.
  • examples/: Example configurations and usage.
  • docs/: Documentation files.

For a detailed breakdown, visit the Structure page.

Development Workflow

To ensure consistency, please follow these guidelines:

  • Branch naming: Use descriptive names like feature/add-auth or bugfix/fix-api.
  • Commit messages: Write clear, concise messages. Example: Fix issue with API response handling.
  • Pull requests: Submit PRs to the main branch. Include a detailed description of changes.
  • Code reviews: Be prepared to address feedback from maintainers.

Coding Standards

All code contributions should adhere to the following:

  • Follow PEP 8 style guide.
  • Use the provided linting tools (flake8).
  • Ensure consistent formatting with black.
  • Write clean, readable code with comments where necessary.

Testing

Before submitting a pull request, ensure all tests pass:

  1. Run the test suite: 
    python -m unittest discover tests
  2. Add tests for new features or bug fixes.
  3. Maintain at least 80% test coverage.

Building and Running

To run the development server:

flask run

For production builds:

export FLASK_ENV=production
flask run

Make sure to configure the necessary environment variables.

Debugging

Use the following tools and techniques for debugging:

  • Enable Flask debug mode (export FLASK_ENV=development).
  • Use logging for tracking issues (import logging).
  • Inspect API responses with tools like Postman or curl.

Submitting Contributions

When ready to submit your work:

  • Ensure all tests pass.
  • Follow the PR checklist:
    • Tests passed
    • Documentation updated
    • Code reviewed
  • Include detailed documentation for any changes.
  • Submit the PR and wait for maintainer review.

Getting Help

If you have questions or need assistance:

  • Visit the FAQ page.
  • Join the discussion on our GitHub Issues page.
  • Contact the maintainer via email listed in the repository.
