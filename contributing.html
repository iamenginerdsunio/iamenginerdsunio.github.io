Contributing

Quick Setup

  1. Clone the repository: git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Navigate to the project directory: cd LibManagement
  3. Create and activate a virtual environment: 
    
python -m venv venv
source venv/bin/activate  # On macOS/Linux
venv\Scripts\activate   # On Windows
  4. Install dependencies: pip install -r requirements.txt
  5. Run the Flask application: python app.py

Contribution Workflow

  1. Fork the repository and clone your fork locally.
  2. Create a new branch for your feature or fix: git checkout -b feature-name
  3. Make your changes, ensuring code adheres to the existing style and conventions.
  4. Run tests to verify your changes: 
    
pytest tests/
  5. Commit your changes with a clear message: git commit -m "Descriptive commit message"
  6. Push your branch: git push origin feature-name
  7. Open a pull request on GitHub against the main branch.

Code Hygiene

  • Follow PEP 8 standards for Python code formatting.
  • Keep functions and methods focused and concise.
  • Document new modules, functions, and classes with clear comments or docstrings.
  • Organize imports into logical groups (standard library, third-party, local modules).
  • Remove unused code and dependencies.

Testing Guidelines

Testing is essential for maintaining the integrity of the application. Ensure your contributions are covered by tests:

  • Add tests for any new features or bug fixes in tests/test_app.py.
  • Run the test suite locally before submitting your pull request.
  • Check that your tests cover edge cases and fail gracefully.

Operational Notes

  • Ensure the application runs properly after your changes by testing with python app.py.
  • Keep the requirements.txt file updated when adding new dependencies.
  • If you modify templates, verify their rendering via the browser and Flask logs.
  • Follow the repository’s branching and pull request conventions.
