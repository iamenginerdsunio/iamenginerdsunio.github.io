Contributing to the Library Management System

Welcome

Thank you for your interest in contributing to the Library Management System! Contributions from the community help us improve and grow the project. Whether you're fixing bugs, creating new features, or improving documentation, your input is highly valued.

Please make sure to adhere to our Code of Conduct to maintain a respectful and inclusive environment.

Prerequisites

  • Tools: Python 3.8 or higher, Flask, SQLAlchemy
  • System Requirements: Compatible with Linux, macOS, and Windows
  • Account Requirements: A GitHub account to submit issues and pull requests

Development Setup

  1. Clone the repository: 
    git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Install dependencies: 
    pip install -r requirements.txt
  3. Configure the environment: Ensure the database URI in app.py is correctly set, e.g.,: 
    SQLALCHEMY_DATABASE_URI = 'sqlite:///library.db'
  4. Verify installation: Run the development server: 
    python app.py
    Visit http://127.0.0.1:5000 to confirm the application is running.

If you encounter issues, consult the project's README or open an issue for help.

Project Structure

The Library Management System follows a modular structure:

  • app.py: Entry point for the application
  • models.py: Database models using SQLAlchemy
  • extensions.py: Initializes shared extensions
  • tests/: Contains unit tests
  • templates/: HTML templates

For more details, visit the Structure page.

Development Workflow

  • Branch Naming: Use descriptive names like feature/add-book-api or bugfix/fix-loan-calculation.
  • Commit Messages: Follow the format: type(scope): message, e.g., feat(api): add endpoint for book retrieval.
  • Pull Requests: Submit PRs with a clear description of changes. Include screenshots or logs if applicable.
  • Code Reviews: Expect maintainers to review and suggest improvements.

Coding Standards

  • Style Guide: Follow PEP 8 guidelines.
  • Linting: Use tools like flake8 to check for errors.
  • Formatting: Ensure consistent formatting using black.
  • Best Practices: Write modular, reusable, and testable code.

Testing

  1. Run tests: 
    pytest tests/
  2. Write new tests: Place test files in the tests/ directory. Use descriptive function names.
  3. Test Coverage: Aim for at least 80% coverage.
