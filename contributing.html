Contributing

Quick Setup

To get started, clone the repository and ensure your environment meets the requirements:

  1. Clone the repository: git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Navigate to the project directory: cd LibManagement
  3. Install dependencies listed in requirements.txt: pip install -r requirements.txt

Contribution Workflow

To contribute to the project, follow these steps:

  1. Fork the repository on GitHub.
  2. Create a new branch for your feature or bugfix: git checkout -b feature-name.
  3. Make your changes in the appropriate files. Refer to the repository structure and architecture documentation as needed.
  4. Commit your changes with a meaningful message: git commit -m "Describe your changes".
  5. Push your branch to your fork: git push origin feature-name.
  6. Create a pull request on the main repository and provide a clear description of your changes.

Code Hygiene

Ensure your changes adhere to the following code hygiene guidelines:

  • Maintain consistent formatting and use meaningful variable names.
  • Follow Python's PEP 8 style guide where applicable.
  • Keep functions and methods concise and focused on single responsibilities.
  • Document new code with inline comments and update relevant documentation files.

Testing Cues

All changes should be validated using the existing pytest test suite before submission:

  • Run tests with pytest tests/.
  • Add new tests for any added functionality in tests/test_app.py.
  • Ensure all tests pass and coverage is maintained or improved.

Operational Notes

Keep in mind the following operational aspects:

  • Core application logic resides in app.py. Review its structure before making changes.
  • Database models are defined in models.py. Ensure compatibility with existing models when extending functionality.
  • Templates use Jinja2 inheritance. Update base.html carefully if modifying shared layout components.
  • Static assets are served from the static/ directory. Place new assets appropriately.
