Contributing to LibManagement

Quick Setup

To set up the development environment, clone the repository and install the dependencies listed in requirements.txt:

git clone https://github.com/iamenginerdsunio/LibManagement.git
cd LibManagement
pip install -r requirements.txt

Contribution Workflow

Follow these steps to contribute:

  1. Fork the repository and create a new branch for your changes.
  2. Make your modifications, ensuring they align with the project's code structure and style.
  3. Write or update tests in tests/test_app.py to cover your changes.
  4. Run all tests with pytest to validate functionality:
    5. pytest
  5. Submit a pull request with a clear description of your changes.

Code Hygiene

Maintain clean and readable code by adhering to these guidelines:

  • Follow Python's PEP 8 coding standards.
  • Organize routes and views logically within app.py.
  • Use meaningful names for functions, variables, and models.
  • Keep templates structured and extend base.html where applicable.

Testing

Tests are located in tests/test_app.py. Ensure that:

  • All routes and workflows (index, books, members, loans) are covered.
  • HTTP responses and expected behaviors are verified.
  • New features include corresponding tests.

Operational Notes

The application uses these core technologies:

  • Flask: For routing and template rendering.
  • Flask-SQLAlchemy: For ORM-based database interactions.
  • Flask-Migrate: To manage database migrations.
  • pytest: For testing.

Ensure compatibility with these dependencies when making changes.
