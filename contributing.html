Contributing to WebAppWithDatabaseDemo

Welcome!

Thank you for your interest in contributing to WebAppWithDatabaseDemo. Your contributions are greatly valued and help make this project better for everyone. Whether you're fixing bugs, adding features, or improving documentation, your input is appreciated.

Please make sure to follow the Code of Conduct to ensure a positive and productive environment for all contributors.

Prerequisites

  • Install Visual Studio (2022 or later) with ASP.NET Core workloads.
  • Ensure .NET SDK version 6.0 or higher is installed.
  • Have a GitHub account to submit pull requests.

Development Setup

Step-by-step instructions

  1. Clone the repository: 
    git clone https://github.com/HoussemDellai/WebAppWithDatabaseDemo.git
  2. Install dependencies: 
    dotnet restore
  3. Configure the environment:

    Update database connection strings in WebApp/appsettings.Development.json.

  4. Verify installation: 
    dotnet build

Troubleshooting

If you encounter issues, check that your .NET SDK and Visual Studio versions meet the prerequisites.

Development Workflow

Branch naming conventions

  • Use descriptive names like feature/add-login or bugfix/fix-homepage.

Commit message format

  • Start with a short description.
  • Include detailed information in the body if necessary.
  • Example: Fix homepage crash when loading employee data.

Pull request process

  1. Create a pull request from your branch.
  2. Ensure all checks pass on GitHub.
  3. Request a review from maintainers.

Coding Standards

Style guide

Follow Microsoft's C# coding guidelines.

Linting setup

Ensure dotnet analyzers are enabled in your IDE.

Testing

Running tests

Writing new tests

Add test methods in the WebApp.Tests project.

Building and Running

Development server

Environment variables

Ensure ASPNETCORE_ENVIRONMENT is set to Development.
