Contributing to Flask-AppBuilder

Welcome to Flask-AppBuilder!

Thank you for considering contributing to Flask-AppBuilder! Your contributions help us improve this framework and provide developers with a powerful tool for building web applications.

Please ensure you adhere to the project's Code of Conduct while contributing.

Prerequisites

  • Required Tools: Python 3.6+ installed on your system.
  • System Requirements: Ensure you have pip and virtualenv installed.
  • Account Requirements: A GitHub account is required for submitting issues and pull requests.

Development Setup

  1. Clone the Repository: 
    git clone https://github.com/dpgaspar/Flask-AppBuilder.git
  2. Install Dependencies: 
    cd Flask-AppBuilder
pip install -r requirements/base.txt
  3. Configure Environment: Set up a virtual environment for development: 
    python -m venv venv
source venv/bin/activate
  4. Verify Installation: Run the tests to ensure everything is working: 
    pytest

Troubleshooting

If you encounter issues during setup, ensure all dependencies are installed correctly and your Python version is compatible.

Development Workflow

  • Branch Naming Conventions: Use descriptive branch names, e.g., feature/add-authentication or bugfix/fix-routing.
  • Commit Messages: Follow this format: 
    [Type]: Short description

Detailed description (if necessary)
Fixes #issue_number
  • Pull Request Process: Open a pull request against the main branch. Ensure your PR includes clear descriptions and references any related issues.
  • Code Reviews: Expect thorough reviews from maintainers. Address all comments promptly.

Coding Standards

Adhere to the following coding standards:

  • Style Guide: Follow PEP 8 for Python coding conventions.
  • Linting: Install and configure flake8: 
    pip install flake8
  • Formatting: Use black for code formatting: 
    pip install black
black .
  • Best Practices: Write modular and reusable code, and ensure proper comments and docstrings.

Testing

  • Running Tests: Execute all tests using: 
    pytest
  • Writing New Tests: Include tests for new features or bug fixes in the tests/ directory.
  • Test Coverage: Aim for 100% test coverage.
  • Organizing Tests: Group tests by functionality (e.g., API, models, views).

Building and Running

  • Development Server: Run the server locally using: 
    flask run
  • Build Process: Flask-AppBuilder does not require a build step but ensure dependencies are up-to-date.
  • Environment Variables: Configure environment variables for your development setup.
  • Hot Reload: Enable Flask's debugging mode for live reloads during development.

Debugging

  • Debugging Tools: Use Flask's built-in debugger for troubleshooting.
  • Common Scenarios: Check logs and tracebacks for errors in routing or database queries.
  • Logging: Configure Flask logging for detailed error tracking.

Submitting Contributions

Follow these guidelines when submitting contributions:

  • Issue Guidelines: Provide detailed descriptions and steps to reproduce bugs or feature requests.
  • Pull Request Checklist: Ensure the following before submitting:
    • Code is linted and formatted.
    • Tests are included and passed.
    • Documentation is updated.
  • Documentation Requirements: Update relevant pages for new features.
  • Review Expectations: Maintain patience during the review process.

Getting Help

  • Questions: Use the GitHub Issues page for queries.
  • Communication Channels: Join discussions on the project's GitHub Discussions tab.
  • Maintainer Contact: Reach out to maintainers for urgent queries.
