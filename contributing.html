Contributing to Flask-AppBuilder

Welcome!

Thank you for your interest in contributing to Flask-AppBuilder! We deeply value the efforts of our contributors, as they help improve the framework for developers worldwide. Whether you’re fixing bugs, writing documentation, or adding new features, your input makes a difference.

Please adhere to our Code of Conduct to ensure a respectful and inclusive environment.

Prerequisites

  • Python (version 3.7 or higher)
  • Git
  • Basic knowledge of Flask and SQLAlchemy
  • A GitHub account

Development Setup

  1. Clone the Repository

    git clone https://github.com/dpgaspar/Flask-AppBuilder.git

  2. Install Dependencies

    cd Flask-AppBuilder
pip install -r requirements.txt

  3. Configure Environment

    Ensure your database is properly configured. Update environment variables as needed:

    export FLASK_APP=app.py
export FLASK_ENV=development

  4. Verify Installation

    Run the development server to ensure everything is working:

    flask run

Development Workflow

  • Follow branch naming conventions: feature/your-feature-name, bugfix/issue-number.
  • Use clear commit messages: Fix issue with API route.
  • Submit pull requests against the main branch.
  • Address all code review comments promptly.

Coding Standards

Maintain clean and consistent code:

  • Follow Python’s PEP 8 style guide.
  • Use the provided pylint configuration for linting.
  • Format code using black.

Testing

Testing is crucial for reliability:

  • Run tests using python -m unittest discover.
  • Write new tests for any added functionality.
  • Maintain high test coverage (above 90%).

Building and Running

Use the following commands to build and run the application:

# Start the development server
flask run

# Build for production
python setup.py sdist

Debugging

Tools and tips:

  • Use Flask’s built-in debugger.
  • Log issues using Python’s logging module.

Submitting Contributions

Before submitting your contributions:

  • Follow issue guidelines.
  • Complete the pull request checklist.
  • Update relevant documentation.

Expect reviews within 3–5 business days.

Getting Help

If you need assistance, feel free to:

  • Ask questions on GitHub Discussions.
  • Reach out to maintainers via email.
