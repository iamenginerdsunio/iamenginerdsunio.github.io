Contributing to LibManagement

Welcome

Thank you for considering contributing to LibManagement! Your contributions help make this Flask-based library management system better for everyone. Whether you're fixing bugs, improving documentation, or introducing new features, your efforts are greatly appreciated.

Please ensure you follow our Code of Conduct to maintain a collaborative and respectful environment.

Prerequisites

  • Python 3.8 or later
  • pip package manager
  • A GitHub account

Development Setup

  1. Clone the repository:

    git clone https://github.com/iamenginerdsunio/LibManagement.git

  2. Install dependencies:

    pip install -r requirements.txt

  3. Set environment variables:

    • FLASK_ENV: Set this to development.

  4. Run the application:

    python app.py

Project Structure Orientation

Heres a brief overview of the main directories:

  • templates/: Contains HTML templates for rendering views.
  • static/: Stores static assets like CSS and JavaScript files.
  • tests/: Houses unit and integration tests for the application.
  • docs/: Provides documentation for setup and usage.

Refer to the Structure page for more details.

Development Workflow

  • Branch Naming: Use descriptive names such as feature/add-user-authentication or bugfix/fix-database-error.
  • Commit Messages: Follow the format [Type] Short Description. Example: [Feature] Add user login functionality.
  • Pull Requests: Ensure your PR contains a detailed description of your changes and references relevant issues.
  • Code Reviews: Be open to constructive feedback and make necessary adjustments.

Coding Standards

  • Style Guide: Adhere to PEP 8 for Python code.
  • Linting: Use flake8 to identify and fix coding issues.
  • Formatting: Apply black for consistent code formatting.

Testing

  • Running Tests: Execute pytest tests/ to run the test suite.
  • Writing Tests: Add new test cases to the tests/ directory, organized by feature or module.

Submitting Contributions

  • Issue Guidelines: Provide clear and detailed descriptions of the problem, including steps to reproduce it.
  • Pull Request Checklist: Ensure all tests pass and relevant documentation is updated before submitting.
  • Documentation: Update relevant sections of the documentation when introducing new features.

Getting Help

If you have any questions or need assistance, you can:

  • Open an issue on our GitHub repository.
  • Start a discussion on GitHub to get feedback from the community.
