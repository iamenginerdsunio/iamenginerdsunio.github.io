Contributing to LibManagement

Quick Setup

To set up the project for development, follow these steps:

  1. Clone the repository:
    2. git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Navigate to the project directory:
    3. cd LibManagement
  3. Create a virtual environment and install dependencies:
    4. python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
  4. Run the Flask application:
    5. python app.py

Contribution Workflow

We welcome contributions to improve the LibManagement project. Please follow these guidelines:

  • Fork the repository on GitHub.
  • Create a new branch for your feature or bugfix:
    • git checkout -b feature/your-feature-name
  • Make your changes and commit them:
    • git add .
git commit -m "Description of your changes"
  • Push your branch to your fork:
    • git push origin feature/your-feature-name
  • Submit a pull request to the main branch of this repository.

Code Hygiene

We maintain the following code hygiene standards:

  • Follow Python's PEP 8 guidelines for code formatting.
  • Write clear, concise commit messages.
  • Document new functions, classes, and modules with comments or docstrings.
  • Keep functions and methods small and focused.

Testing

Automated tests are crucial for maintaining project stability. The project uses pytest and pytest-flask for testing. To run the tests:

pytest

Add tests for any new features or bug fixes in the tests directory. Ensure all tests pass before submitting a pull request.

Operational Notes

Key operational details to keep in mind:

  • The entry point for the application is app.py.
  • Templates are organized by domain in the templates directory (e.g., templates/books/, templates/members/, templates/loans/).
  • Static assets are located in the static directory.
