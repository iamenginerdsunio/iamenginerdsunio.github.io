Contributing Guide

Quick Setup

To set up the project locally, follow these steps:

  1. Clone the repository: git clone https://github.com/iamenginerdsunio/LibManagement.git
  2. Navigate to the project directory: cd LibManagement
  3. Create a virtual environment: python -m venv venv
  4. Activate the virtual environment:
    • On Windows: venv\Scripts\activate
    • On macOS/Linux: source venv/bin/activate
  5. Install dependencies: pip install -r requirements.txt
  6. Run the Flask application: python app.py

Contribution Workflow

To contribute to the project, follow this workflow:

  1. Fork the repository to your GitHub account.
  2. Create a new branch for your feature or fix: git checkout -b feature-name
  3. Make your changes in the codebase.
  4. Ensure the application runs successfully and passes all tests.
  5. Commit your changes with a clear message: git commit -m "Add feature description"
  6. Push the changes to your fork: git push origin feature-name
  7. Create a pull request against the main branch of the original repository.

Code Hygiene

When contributing, ensure the following:

  • Follow PEP 8 standards for Python code.
  • Write clear and descriptive commit messages.
  • Remove any commented-out or unused code.
  • Use meaningful names for functions, variables, and classes.

Testing Cues

Before submitting a contribution, ensure the following tests pass:

  1. Run the automated tests located in tests/test_app.py using pytest: pytest tests
  2. Test all routes (books, members, loans) for expected behavior and verify CRUD workflows.
  3. Manually verify that templates render correctly and display appropriate data.

Operational Notes

Keep in mind the following operational considerations:

  • Ensure any new dependencies are added to requirements.txt.
  • Document any new routes, models, or features in the appropriate Markdown files within the docs directory (contents not detected).
  • Maintain consistency with existing routes and templates.
  • Check compatibility with the existing Flask-SQLAlchemy and pytest setups.
