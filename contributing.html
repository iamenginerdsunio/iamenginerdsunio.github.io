Contributing to iamenginerdsunio.github.io

Welcome

Thank you for your interest in contributing to iamenginerdsunio.github.io! Contributions help improve the project and ensure its continued success. Please adhere to the project's code of conduct while contributing.

Prerequisites

Before you begin, ensure you have the following:

  • A web browser
  • A text editor for editing HTML and CSS files
  • A GitHub account (optional for cloning the repository)

Development Setup

Follow these steps to set up the project locally:

  1. Clone the repository:

    git clone https://github.com/iamenginerdsunio/iamenginerdsunio.github.io.git

  2. Open the HTML file:

    Open index.html in a web browser

Troubleshooting: If you encounter issues, ensure your browser supports HTML5 and CSS3 standards.

Project Structure Orientation

The repository is organized as follows:

/\n├── index.html\n└── styles.css

Refer to the Structure page for a detailed overview.

Development Workflow

Follow these guidelines during development:

Branch Naming
Use descriptive names, e.g., feature-update-header.
Commit Messages
Write clear, concise messages, e.g., Add responsive design to header.
Pull Request Process
Submit PRs with detailed descriptions and adhere to coding standards.
Code Review
Be open to feedback and incorporate suggested changes.

Coding Standards

Maintain the following standards:

  • Use semantic HTML5 tags.
  • Follow best practices for CSS organization.
  • Ensure code readability and consistency.

Testing

Testing is done manually:

  • Open index.html in various browsers.
  • Ensure consistent rendering and functionality.

Building and Running

The website does not require a build process. Simply open index.html in a browser to view the project.

Debugging

Use browser developer tools for debugging:

  • Inspect elements and styles.
  • Check for console errors.

Submitting Contributions

When submitting contributions:

  • Follow the issue template if reporting bugs or suggesting features.
  • Ensure PRs are complete and well-documented.
  • Expect reviews within a week.

Getting Help

If you have questions, feel free to:

  • Check the Structure page.
  • Contact the repository maintainer via GitHub.
