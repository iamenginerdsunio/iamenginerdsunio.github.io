Contributing Guide

Welcome to LibManagement

Thank you for your interest in contributing to LibManagement, a Flask-based library management system. We highly value contributions from the community, whether you are fixing bugs, adding features, or improving documentation.

All contributors are expected to adhere to our Code of Conduct to maintain a welcoming and collaborative environment.

Prerequisites

  • Python 3.x installed on your system
  • pip for managing Python packages
  • A GitHub account for submitting issues and pull requests

Development Setup

  1. Clone the repository:

    git clone https://github.com/iamenginerdsunio/LibManagement.git

  2. Navigate to the project directory:

    cd LibManagement

  3. Install dependencies:

    pip install -r requirements.txt

  4. Verify the installation:

    python app.py

If you encounter issues during setup, please check your Python and pip versions or refer to our Documentation.

Project Structure Orientation

LibManagement follows a structured layout to ensure maintainability:

  • templates/: Contains HTML templates
  • static/: Stores CSS and JavaScript files
  • tests/: Includes test scripts
  • docs/: Provides project documentation

For a detailed directory breakdown, visit the Structure page.

Development Workflow

Follow these steps for contributing:

  • Use feature-specific branch names (e.g., feature/add-search).
  • Write clear commit messages (e.g., "Fix issue with book search functionality").
  • Submit pull requests with detailed descriptions of changes.
  • Ensure your code passes all tests before requesting a review.

Coding Standards

To maintain consistency:

  • Follow PEP 8 for Python code style.
  • Use the provided linting tools to check your code:
    • flake8 .
  • Format code using:
    • black .

Testing

LibManagement uses Python's unittest framework:

  • Run all tests:
    • python -m unittest discover tests
  • Add tests for new features in the tests/ directory.
  • Ensure test coverage is high by writing comprehensive test cases.

Building and Running

To start the development server:

python app.py

Environment variables required:

  • SQLALCHEMY_DATABASE_URI for database connection

Debugging

Use debugging tools such as:

  • pdb for Python debugging
  • Flask's built-in debugger

Check logs generated by the application for insights into issues.

Submitting Contributions

Before submitting:

  • File an issue for discussion, if necessary.
  • Follow the pull request checklist:
    • Code is formatted and linted.
    • Tests are included and passing.
    • Documentation is updated, if applicable.
  • Expect feedback within a few days.

Getting Help

If you have questions or need assistance:

  • Check the Documentation.
  • Open an issue on GitHub.
  • Contact maintainers listed in the repository.
