Contributing to Flask-AppBuilder

Welcome!

Thank you for considering contributing to Flask-AppBuilder! Your contributions are invaluable to improving the framework and helping the community build better web applications. Please make sure to adhere to the Code of Conduct while contributing.

Prerequisites

  • Python 3.8+ (required)
  • Flask (required)
  • SQLAlchemy (required)
  • GitHub account (required for submitting contributions)

Development Setup

  1. Clone the repository

    Run the following command:

    git clone https://github.com/dpgaspar/Flask-AppBuilder.git

  2. Install dependencies

    Navigate to the project directory and install dependencies:

    cd Flask-AppBuilder
pip install -r requirements.txt

  3. Configure environment

    Create a virtual environment and activate it:

    python -m venv env
source env/bin/activate

  4. Verify installation

    Run the application to verify the setup:

    flask run

Development Workflow

  • Branch naming conventions: Use `feature/your-feature-name` or `bugfix/your-bug-name`.
  • Commit message format: Follow the structure `type(scope): message`. Example: fix(models): resolve null pointer exception.
  • Pull request process: Ensure your code is well-documented and passes all tests before submitting a PR.
  • Code review: Expect reviews within 3-5 business days.

Coding Standards

  • Style guide: Follow PEP 8.
  • Linting setup: Use flake8 for linting.
  • Formatting rules: Use black for consistent formatting.
