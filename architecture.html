Architecture

Overview

The LibManagement application is a Flask-based web application designed for managing books, members, and loans. It demonstrates CRUD operations and core library workflows through a server-rendered web interface.

Key Architectural Features

  • Frameworks: Flask for routing and request handling, Flask-SQLAlchemy for ORM, and Flask-WTF for form handling.
  • Structure: A single-module Flask app (app.py) with models defined in models.py, shared extensions in extensions.py, and Jinja2 templates grouped by domain under templates/.
  • Testing: Routes and flows are validated using a pytest test suite in tests/test_app.py.

Layers

  • Presentation: Jinja2 templates (e.g., base.html, index.html, and domain-specific templates under books, members, and loans directories).
  • Application Logic: Flask routes in app.py manage CRUD operations and workflows for books, members, and loans.
  • Persistence: SQLAlchemy ORM models defined in models.py handle database interaction and relationships between entities.

Framework Relationships

The application integrates Flask for web requests and routing, Flask-SQLAlchemy for ORM functionality, and Flask-WTF for handling form validation and processing. These frameworks are configured centrally in app.py and extensions.py.
