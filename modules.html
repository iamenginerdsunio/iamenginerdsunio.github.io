Module Overview

Total Modules: 2

The core modules are organized to follow the MVC (Model-View-Controller) architecture, ensuring modularity and scalability. Module names align with their functionality, such as AppBuilder for application initialization and BaseView for managing views and routing.

Core Modules

AppBuilder

  • Location: flask_appbuilder/__init__.py
  • Purpose: Initializes and configures the Flask application with SQLAlchemy.
  • Key Functions:
    • __init__(app: Flask, db: SQLAlchemy) -> None: Constructor for the AppBuilder class.
  • Dependencies: Flask, SQLAlchemy
  • Used By: Views and models modules

Code Example

class AppBuilder:
    def __init__(self, app: Flask, db: SQLAlchemy):
        self.app = app
        self.db = db
        self.init_app()

BaseView

  • Location: flask_appbuilder/views.py
  • Purpose: Defines the base class for creating views and routing logic.
  • Key Functions:
    • register_view(view: BaseView) -> None: Registers a view with Flask.
    • render() -> str: Renders the template associated with the view.
  • Dependencies: Flask
  • Used By: Controllers and routes

Code Example

def register_view(view: BaseView) -> None:
    app.add_url_rule(view.url, view_func=view.render)
