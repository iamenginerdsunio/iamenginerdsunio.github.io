Library Management System Documentation

A server-rendered web application for managing books, members, and loans.

Overview

This project is a web application for managing a small library, including books, members, and loans. It provides CRUD operations, searching, and borrowing/return flows through a web UI.

Primary Languages: Python, HTML

Frameworks: Flask, Flask-SQLAlchemy, Bootstrap (via CDN in templates)

Architecture

The application is structured as a single Flask module with organized SQLAlchemy models and Jinja2 templates for books, members, and loans. It uses a shared extensions module for app-wide services and a tests package for automated testing.

Setup

Install dependencies listed in requirements.txt. Use Flask CLI or run app.py directly to start the server.

Code Flow

The app initializes Flask extensions in extensions.py, defines routes in app.py, and uses SQLAlchemy models from models.py. Templates are rendered via Jinja2 for all UI.

Key Modules

  • app.py: Defines routes and logic for CRUD operations.
  • models.py: Contains SQLAlchemy models for books, members, and loans.
  • extensions.py: Initializes shared Flask extensions.
  • templates/: Contains Jinja2 templates for rendering HTML.
  • tests/: Automated tests for application flows.

Data Model

  • Book: Attributes like title and author.
  • Member: Attributes like name and contact info.
  • Loan: Links books and members with loan/return dates.

UI Patterns

  • Base layout with Jinja2 template inheritance.
  • Bootstrap-styled forms and tables.
  • Domain-specific templates for books, members, and loans.
